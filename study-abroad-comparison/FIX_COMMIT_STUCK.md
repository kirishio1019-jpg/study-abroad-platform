# コミットが途中で止まっている場合の対処法

## 🔍 現在の状態

`COMMIT_EDITMSG`ファイルが開いている場合、コミットプロセスの途中で止まっています。

---

## ✅ 解決方法

### ステップ1: コミットメッセージを保存して閉じる

1. **COMMIT_EDITMSGファイルを確認**
   - ファイルが開いている状態で、最初の行にコミットメッセージがあるか確認
   - `#` で始まる行はコメントなので無視

2. **コミットメッセージを入力**（まだの場合）
   - 最初の行にメッセージを入力（例：`Initial commit`）

3. **ファイルを保存して閉じる**
   - `Ctrl + S` で保存
   - `Ctrl + W` でファイルを閉じる
   - または、タブをクリックして閉じる

4. **コミットが完了するのを待つ**
   - 自動的にコミットが完了します

---

### ステップ2: コミットプロセスをキャンセルする場合

もしコミットをキャンセルしたい場合：

1. **COMMIT_EDITMSGファイルを閉じる**
   - `Ctrl + W` で閉じる

2. **ファイルの内容を削除または変更**
   - すべての行を削除して空にする
   - または、`# Please enter the commit message...` の行だけを残す

3. **保存して閉じる**
   - `Ctrl + S` → `Ctrl + W`

これでコミットがキャンセルされます。

---

### ステップ3: コミット完了後、PullとPushを実行

コミットが完了したら：

1. **Pullを実行**
   - `Ctrl + Shift + P` → 「Git: Pull」

2. **Pushを実行**
   - `Ctrl + Shift + P` → 「Git: Push」

---

## 🆘 それでも解決しない場合

### 方法A: Cursorを再起動

1. Cursorを完全に閉じる
2. 再度開く
3. ソース管理パネルを確認

### 方法B: コマンドパレットから直接実行

1. `Ctrl + Shift + P` を押す
2. 「**Git: Close Repository**」と入力して選択
3. 再度リポジトリを開く

---

## 📝 推奨手順

1. ✅ COMMIT_EDITMSGファイルを開く
2. ✅ コミットメッセージを確認/入力
3. ✅ ファイルを保存（`Ctrl + S`）
4. ✅ ファイルを閉じる（`Ctrl + W`）
5. ✅ コミット完了を待つ
6. ✅ Pullを実行
7. ✅ Pushを実行
